# ai Assist Settings Form

## Project Overview

Same **Settings Form with Validation** built twice using two different
prompting approaches, to compare the difference between accepting AI output
blindly vs. directing it with specs and verification.

- **Round 1** (`ROUND1.html`): a single, general prompt, output accepted
  with no review.
- **Round 2** (`ROUND2.html`): a detailed prompt with field-by-field rules,
  edge cases, an accessibility requirement, and a mandatory
  verify-before-done step.

## How to view the forms

Just open in a browser — no server, no build tools, no npm install:
- `ROUND1.html`
- `ROUND2.html`

## Files in this folder

- `README.md` — this file.
- `PROMPT_USED.md` — exact prompts used for both rounds.
- `WORKFLOW.md` — correctness / accessibility / edge case / review-effort
  write-up.
- `CLAUDE.md` — project rules learned from the drill.
- `ROUND1.html` — Round 1 output, self-contained (code included).
- `ROUND2.html` — Round 2 output, self-contained (code included).
- `COMPARISON.md` — final side-by-side comparison of both rounds.
